1/30/2024   
[Safe Network](https://safenetforum.org/t/is-maidsafe-scared-to-launch/37480)  
I have followed this project for a very long time and know some of the contributors, so it causes me some distress to see how the project has gone.
- First, I love the projects ideals and share the stated organizing principles.  The concept of a stable, anonymous and nonmanipulable network that is secure and decentralized by design rather than by the declaration of the builders is attractive. It grants the users space to form ideas, opinions and associations without outside meddling.
- Second, The architecture looks good.  Avoiding the cloud gives the network space to avoid censorship. A malevolent actor would have to either remove all tcpip all traffic or ban encryption to influence the network.  Either of these actions would be an obvious challenge to civil liberties and would pose a potential threat to human rights.
- Third, it is open source.  It is vital that the network code be viewable by anyone.  This allows trust to be built between the user and the network.  A malevolent actor can't hide behind "proprietary" code that is designed to hide behavior.

 #### Analysis:   
 This project needs to be forked.  Given that this project has been active for 14 years and has perpetually been ready to launch "next year" is a huge red flag to me.  

I also find their "moonshot" argument suspicious if not completely deceitful. I have worked on networks and written software for over 25 years and know that protocols are not large, complex bits of code, but rather small and VERY hard to get correct from the design perspective. It is their simplicity that gives them the power. It shouldn't take this long to write the code with the functionality which is already set in the rfc.

The behavior of the community is also concerning. After at least two rounds of funding, with "suggestions" each time that the network will be finished "soon", it is dubious that it will be finished.  Also, why all the concern with the price of the maid coin? This was intended to be a temporary means of funding to get the network up and running, but now it seems that the object is to make money on the fluctuations of the coin price.  Is the price being pumped and dumped? Is that what all the long winded and mostly nonsensical posts in the post linked above? Getting this thing up and running shouldn't be that expensive.  Server costs for the test networks whether on the cloud or native would not be that costly considering that the amount of data and processing involved is not large.  Why is it taking so much money? I have been told by contributors to the network that it is a side project for them. They don't seem to be concerned about completion.  Perhaps some of the above explains why.
